# 执行计划与进度记录

## 约束说明

- 本文件记录可公开的执行计划、关键判断和进度更新；不会记录不可公开的内部推理细节。
- 在读取 `TODO.md` 前，先建立本初始计划文件，以满足“先写计划再执行命令”的要求。
- `TODO.md` 是任务顺序和完成标准的唯一权威来源；`PLAN.md` 仅在阶段级计划变化时更新。
- 本轮只完成第一个未完成任务，完成后提交 Git commit 并停止。

## 初始步骤

1. 读取 `TODO.md`，按标题是否带 `[DONE]` 判断第一个未完成任务。
2. 查看最新提交信息，仅判断是否明确提到与当前任务直接相关的未完成问题。
3. 阅读当前任务涉及的设计文档、源码和测试，确认实现边界。
4. 按仓库现有模式实施当前任务，避免 workaround 或缩窄范围。
5. 先运行 `cargo fmt --all`，再运行 `cargo clippy --all-targets -- -D warnings`，通过后运行相关测试和必要的完整测试。
6. 更新 `TODO.md`：给完成任务标题加 `[DONE]`，补充完成记录与验证记录。
7. 提交本轮所有相关变更，提交信息包含任务编号和明确说明。
8. 停止，不进入下一个任务。
